## STATISTICS NOT FOUND (Use Qualitative Language)

The following claims from the current website could NOT be verified from credible sources:

| Current Claim | Recommendation |
|---------------|----------------|
| "391% increase in conversions from 1-min response" | Use: "Faster response significantly improves conversion" |
| "119% higher click-through rates from automation" | Use: "Higher engagement rates from automated follow-ups" |
| "95% reduction in compliance issues" | Use: "Significantly reduces compliance risks" |
| "85-92% accuracy for AI lead scoring" | Use: "High-accuracy AI lead scoring" |
| "3.7x ROI within 90 days" | REMOVE - fabricated claim |
| "90% user adoption in 30 days" | REMOVE - fabricated claim |
| "30 hours per transaction on admin tasks" | Use: "Significant time on transaction admin" |
| "$15,000+ annual software spend per agent" | Use: "Significant annual software costs" (NAR says 24% spend $500+/month = $6,000+/year) |
| "62% of leads fall through cracks" | Use: "Many leads go uncontacted" (27% contacted is verified) |

## USAGE GUIDELINES

### For Data Files (modules.ts, market-stats.ts, etc.)

When updating statistics, use this format:
```typescript
{
  stat: "68%",
  context: "of agents have adopted AI",
  source: "NAR 2025 Technology Survey",
  sourceUrl: "https://nar.realtor/research-and-statistics/research-reports/realtor-technology-survey",
  year: 2025
}
```

### For Qualitative Replacements

| Instead of... | Use... |
|---------------|--------|
| "X% more deals" | "More deals closed" |
| "X% accuracy" | "High accuracy" |
| "X% time savings" | "Significant time savings" |
| "X% higher engagement" | "Higher engagement rates" |
| Specific ROI claims | "Strong ROI" or remove |

### Citation Best Practices

1. Always include source URL
2. Note publication year
3. Prefer Tier 1 sources (NAR, Gartner, McKinsey)
4. For ranges, use the conservative end
5. Mark projections/forecasts as such
